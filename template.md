---

## 头部模板

```markdown
<callout emoji="writing_hand" background-color="light-orange">
**茶话会时间**：2026 年 X 月 X 日 XX:XX-XX:XX（X 小时 X 分钟）
**参与人员**：<mention-user id="ou_xxx"/>...

```

## 问题模板

```markdown
## 问题 X：[问题标题]（X 票）

<quote-container>
**问题背景**：[1-2 句话描述问题背景]
</quote-container>

<callout emoji="pushpin" background-color="light-orange">
**核心结论**：[1-2 句话总结核心结论]
**王爷原话**：
- "[关键原话 1]"
- "[关键原话 2]"
</callout>

### [子问题/分论点]

[内容按"战略→政策→当下"逻辑组织]

- **总结句**：原话嵌入正文，作为论据支撑...

<quote-container>
案例：[案例内容]
</quote-container>
```

## 提炼检查清单

### 结构检查
- [ ] 问题按票数降序排列
- [ ] 每个问题有核心结论框（callout + pushpin）
- [ ] 子问题层级不超过 4 层
- [ ] 内容按逻辑顺序组织（非时间顺序）

### 内容检查
- [ ] 删除所有时间戳（如 23:13、31:19-33:55）
- [ ] 删除所有发言人标注（如"何家雄提问"、"黎文祥阐述"）
- [ ] 删除口语化表达（"啊"、"那个"、"是吧"、"OK"）
- [ ] 删除寒暄和过渡语（"好，我们休息一下"）
- [ ] 原话≤20%，总结≥80%

### 格式检查
- [ ] 核心结论加粗前置
- [ ] 原话嵌入正文（非独立引用块）
- [ ] 案例使用 quote-container 标注
- [ ] 待办事项明确（责任部门 + 时间）

### 字数检查
- [ ] 3 小时会议→10k-12k 字
- [ ] 压缩比 2-3 倍
- [ ] 每个问题字数合理分配（重点问题多写，次要问题少写）

## 快速参考

### 原话处理 5 原则
1. **必要性**：只引用无法更好总结的内容
2. **嵌入性**：原话嵌入正文，作为论据支撑
3. **精简性**：删除口语化表达
4. **上下文性**：金句需要有解释和上下文
5. **比例性**：原话≤20%，总结≥80%

### 压缩比例参考
| 问题类型 | 原始字数 | 最终字数 | 压缩比 |
|:---|:---|:---|:---|
| 战略问题 | ~8,000 | ~2,500 | 3.2:1 |
| 政策问题 | ~7,500 | ~2,200 | 3.4:1 |
| 执行问题 | ~3,500 | ~400 | 8.8:1 |

### 层级控制原则
- **基于内容决定**：不是刻意控制层数，而是根据内容复杂度自然形成
- **内容复杂**：可以采用 4 层结构（战略→政策→当下→原则）
- **内容简单**：2 层结构即可（核心原则→具体做法）
